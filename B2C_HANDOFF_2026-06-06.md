## Поточні функції прототипу

- Єдиний блок `Продаж`, який замінив `POS / чеки / каса`.
- `Продаж` має пошук товару за назвою, SKU, штрихкодом або QR.
- Перед проведенням продажу обов'язкове модальне підтвердження методу оплати.
- Якщо оплата `Банк`, має бути вибраний клієнт з довідника, не `Роздрібний покупець`.
- Клієнт у продажі вибирається тільки через поле пошуку, без окремого списку покупців.
- Списки продажів, повернень і KPI-розшифровок зроблені рядками, зі сортуванням, пошуком і згортанням.
- KPI-картки клікабельні за дозволом ролі.
- Повернення по чеку уточнює модальним вікном спосіб повернення грошей: POS або готівка.
- Інвентаризація має додавання товарів із довідника, кнопку `додати всі товари із залишками`, ручне додавання після заповнення, видалення рядків, фактичні залишки, друк листа, barcode/QR сканування, пересорт і проведення, яке змінює залишки.
- Працівники мають ролі `Директор`, `Адміністратор`, `Продавець`, `Касир`.
- Права ролей редагуються галочками; блоки і дії без дозволу приховуються.
- `Обмін даними` показує імпорт, експорт, перебіг процесу, журнал обміну і автоматизацію.
- У `Клієнти і лояльність` додано створення нового B2C-клієнта.
- Новий B2C-клієнт одразу доступний у продажі і чекає експорту в `one_c_mirror.b2c_counterparties_export_queue`.
- Додано перший server-backed режим: Node.js сервер віддає app, `/api/state`, `/api/settings`, `/api/bootstrap`, а дані зберігаються в `data/` на сервері.
- Додано блок `Налаштування` з параметрами зовнішнього доступу, server storage, multi-user режиму й автооновлення.

## Як запустити

```powershell
cd D:\Codex\CRM\retail-crm
powershell -ExecutionPolicy Bypass -File .\serve-lan.ps1 -Port 8790
```

Для реального багатокористувацького server-backed режиму:

```powershell
cd D:\Codex\CRM\retail-crm
powershell -ExecutionPolicy Bypass -File .\start-server.ps1 -HostName 0.0.0.0 -Port 8790 -PublicHost 192.168.0.5 -DataDir data
```

`start-server.ps1` спочатку пробує Node.js, а якщо Node не підтримує стару Windows на сервері, автоматично запускає `server.ps1` як PowerShell TCP server.

Основна адреса:

```text
http://192.168.0.5:8790/index.html
```

Якщо на сервері немає Git, з локального ПК:

```powershell
cd D:\Codex\CRM\retail-crm
$cred = Get-Credential -UserName '192.168.0.5\zahar'
powershell -ExecutionPolicy Bypass -File .\deploy-remote.ps1 -ComputerName 192.168.0.5 -Credential $cred -RemotePath D:\Codex\CRM\retail-crm -Port 8790 -PublicHost 192.168.0.5
```

Якщо на сервері також немає Node.js:

```powershell
powershell -ExecutionPolicy Bypass -File .\deploy-remote.ps1 -ComputerName 192.168.0.5 -Credential $cred -RemotePath D:\Codex\CRM\retail-crm -Port 8790 -PublicHost 192.168.0.5 -IncludeBundledNode
```

## Типовий цикл роботи

1. Прочитати цей handoff і останні записи `logs/B2C_WORK_LOG_2026-06-06.md`.
2. Перевірити `git status --short --branch`.
3. Виконати зміну в `retail-crm`.

7. Записати команди, відповіді і кодові зміни в B2C log.
8. `git add`, `git commit`, `git push`.
9. Додати результат push у B2C log окремим log-only commit і знову push.
