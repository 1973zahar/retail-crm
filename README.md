# B2C Retail CRM

Окремий проєкт для блоку роздрібної торгівлі.

Цей проєкт не є частиною `marketplace-crm` і не змінює блоки маркетплейсу або B2B.

## Структура

- `index.html` - окрема сторінка B2C роздрібного магазину.
- `styles.css` - стилі тільки для retail-блоку.
- `app.js` - клієнтська логіка роздрібного магазину.
- `logs/` - окремі робочі логи B2C.

## Поточний прототип

MVP-прототип містить:

- панель KPI магазину;
- новий чек з кошиком на кілька позицій;
- пошук товару за назвою, SKU або штрихкодом;
- автоматичну знижку лояльності за карткою клієнта;
- журнал чеків;
- друкований перегляд чека;
- повне та часткове повернення по чеку;
- довідник товарів тільки з SQL-імпорту, без ручного створення SKU;
- довідник клієнтів з рівнями лояльності;
- залишки магазину;
- надходження в магазин тільки з SQL-імпорту, без ручного оприбуткування;
- відкриття і закриття касової зміни;
- денний звіт;
- внутрішній журнал операцій.

Відкрити локально можна через `index.html`.

## GitHub

Ця папка має бути окремим git/GitHub репозиторієм. Локальний git ініціалізується окремо всередині `retail-crm`, без зв'язку з `marketplace-crm`.

## LAN переглядалка

Щоб відкрити B2C-прототип з іншого комп'ютера або телефона в тій самій мережі:

```powershell
cd D:\Codex\CRM\retail-crm
powershell -ExecutionPolicy Bypass -File .\serve-lan.ps1 -Port 8790
```

У виводі скрипт покаже локальну адресу та LAN-адресу. На іншому пристрої відкрий:

```text
http://LAN-IP:8790/index.html
```

Якщо Windows Defender Firewall покаже запит, дозволь доступ для приватної мережі.
